# Changelog Policy

## Versioning

- Use semantic versioning for published skill releases.
- Patch releases for documentation, metadata, and backward-compatible template updates.
- Minor releases for additive references, templates, or integrations.
- Major releases for structural or behavioral changes that may affect downstream consumers.

## Release Notes

Each release should summarize:
- new references
- new templates
- changed integrations
- breaking changes, if any
